## §1 需求背景与目标 [必填]

- **背景**：当前平台需要复刻公司内部自动化测试平台中“环境通过率”和“模块通过率”两个核心页面，用于查看每天定时执行后的接口自动化测试结果、定位失败用例，并通过 Jenkins 触发失败重试或模块重试。
- **目标**：
  - 每天凌晨 2 点通过 Jenkins 定时执行全部测试用例。
  - 在环境维度展示当前测试环境的整体通过率和执行概况。
  - 在模块维度展示每个模块的最新执行结果、通过率、负责人和操作入口。
  - 点击模块通过率后展示当前模块全部测试用例，默认筛选失败用例，支持失败、通过、跳过三种状态筛选。
  - 支持一键失败重试和模块重试，并严格区分两类重试对“日期”和“执行时间”的更新规则。
- **成功指标 / 价值**：
  - 测试执行完成后，平台能按环境和模块稳定展示最新结果。
  - 模块通过率公式统一为：`(总用例数 - 失败用例数) / 总用例数`。
  - 主人可通过“日期”判断测试是否发生在开发代码构建之后。
  - 失败用例重试不污染模块完整执行时间，模块重试能刷新完整模块执行基线。

## §2 范围 [必填]

- **做（in scope）**：
  - 环境通过率主页面：环境下拉、环境汇总表、环境通过率统计。
  - 模块通过率页面：环境筛选、模块列表、通过率详情、状态筛选和操作按钮。
  - 每日凌晨 2 点全量测试结果的数据展示规则。
  - 一键失败重试：重试当前模块全部失败用例。
  - 模块重试：重跑当前模块全部用例，并更新模块“日期”和“执行时间”。
  - 通过率详情弹窗内支持勾选失败用例后重试，并支持弹窗内一键失败重试。
  - 近 7 天、近 30 天测试数据入口和模块级趋势弹窗。
  - 当前模块当天 Jenkins 任务弹窗、取消任务、查看 Allure 报告和查看 Jenkins 任务。
  - 环境通过率页保留“生成环境报告”按钮作为后期 AI 分析报告入口占位。
  - 前置用户管理设计：管理人员、普通成员、邀请码注册和一次性邀请码使用规则。
  - 基于 `api-test/utils/package_module.yaml` 的模块元数据读取与入库。
  - DRF API 契约草案、数据库表设计、状态机和容器化兼容检查。
- **不做（out of scope）**：
  - 暂不探索或设计主人未指定的其它旧平台页面。
  - 暂不实现旧平台中的上传报告、环境比对、作废、重启数据库、增加模块记录等扩展操作。
  - 暂不提交后端、前端、Jenkins 或 `api-test` 代码实现。
  - 暂不固化真实内网 Jenkins URL、真实账号、token、cookie 或敏感业务地址。

### 注册与登录规则

- 系统不允许自由注册。
- 注册必须填写未使用、未过期、未作废的邀请码。
- 邀请码只能由管理人员在后台生成并分享。
- 一个邀请码只能成功使用一次，使用后立即绑定注册用户并进入 `used` 状态。
- 首个管理人员账号由初始化脚本或后台种子数据创建，不通过公开注册生成。
- 用户角色仅包含 `admin`（管理人员）和 `member`（普通成员）；本需求不引入多租户、项目级权限和更细粒度 RBAC。

## §4 功能清单与验收标准 [必填 · 核心章节]

### F1 模块元数据读取

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 平台从 `api-test/utils/package_module.yaml` 读取模块配置。
  - YAML 顶层 key 作为“用例包名”，例如 `test_gbif_case`。
  - `module_name` 作为“模块名称”。
  - `module_dev` 作为“模块开发”。
  - `module_test` 作为“模块测试”。
  - 页面字段以 YAML 为基础展示，不在前端硬编码模块名称或负责人。
- **关联数据表**：`test_module`
- **验收标准**：
  - `AC1.1` — Given `package_module.yaml` 存在模块配置 When 系统同步模块元数据 Then 每个顶层 key 都生成或更新一条启用模块记录。
  - `AC1.2` — Given YAML 中模块负责人发生变化 When 下一次同步执行 Then 模块通过率页展示最新负责人。
  - `AC1.3` — Given YAML 缺少必填字段 When 同步执行 Then 记录同步失败原因，不能生成字段不完整的模块记录。

### F2 每日凌晨 2 点全量执行

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - Jenkins 每天凌晨 2 点触发全量测试 Job。
  - Jenkins 调用 `api-test` 执行全部模块用例。
  - 执行完成后，DRF 同步环境汇总、模块快照、用例结果和 Jenkins 任务记录。
  - 每日全量执行会更新模块“日期”和“执行时间”。
- **关联数据表**：`test_run`、`environment_snapshot`、`module_snapshot`、`test_case_result`、`jenkins_task`
- **验收标准**：
  - `AC2.1` — Given 当前时间到达每天 02:00 When Jenkins 定时任务触发 Then 平台产生一条全量执行任务记录。
  - `AC2.2` — Given 全量执行成功完成 When DRF 同步结果 Then 环境通过率页和模块通过率页展示最新统计。
  - `AC2.3` — Given 某模块全量执行完成 When 查看模块列表 Then “日期”为该模块本次完整执行完成时间，“执行时间”为该模块全部用例耗时。
  - `AC2.4` — Given Jenkins 执行失败 When DRF 同步任务状态 Then 对应任务标记为失败，并保留可诊断的错误摘要。

### F3 环境通过率主页面

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 页面提供测试环境下拉。
  - 当前初始测试环境仅一个：展示名“模拟测试环境”，基础地址 `https://api.gbif.org`。
  - 选择环境后展示该环境最近一次全量执行或最新有效执行的整体统计。
  - 环境通过率按环境维度聚合模块统计。
  - 页面保留“生成环境报告”按钮。本期点击后只提示“AI 分析报告功能后续实现”，不触发 Jenkins、不写入报告数据、不创建后台任务。
- **关联数据表**：`test_environment`、`environment_snapshot`
- **验收标准**：
  - `AC3.1` — Given 系统已配置“模拟测试环境” When 打开环境通过率页 Then 下拉包含“模拟测试环境”。
  - `AC3.2` — Given 选择“模拟测试环境” When 查询环境通过率 Then 表格展示环境、开始时间、结束时间、环境地址、通过率、运行时间和操作入口。
  - `AC3.3` — Given 暂无执行结果 When 选择环境 Then 页面展示空状态，不报前端错误。
  - `AC3.4` — Given 环境地址属于配置项 When 页面展示 Then 地址来自后端授权响应，不在前端代码硬编码。
  - `AC3.5` — Given 用户点击“生成环境报告” When 本期 AI 分析报告未实现 Then 页面展示后续实现提示，不触发任何 Jenkins 任务。

### F4 模块通过率主页面

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 页面按模块展示测试结果。
  - 重点字段：日期、用例包名、模块名称、模块开发、模块测试、通过率、执行时间、操作按钮。
  - 通过率公式固定为：`(总用例数 - 失败用例数) / 总用例数`。
  - 失败用例数只统计当前状态为“失败”的用例；跳过不计入失败数，但仍计入总用例数。
  - “日期”和“执行时间”只在每日全量执行和模块重试后更新；失败重试不更新这两个字段。
- **关联数据表**：`module_snapshot`、`test_module`
- **验收标准**：
  - `AC4.1` — Given 模块存在最新快照 When 打开模块通过率页 Then 表格展示该模块的日期、用例包名、模块名称、模块开发、模块测试、通过率和执行时间。
  - `AC4.2` — Given 模块总用例数为 100、失败用例数为 4 When 页面展示 Then 通过率为 96.00%。
  - `AC4.3` — Given 模块刚完成失败重试 When 查看模块行 Then 通过率可更新，但日期和执行时间保持失败重试前的完整执行值。
  - `AC4.4` — Given 模块刚完成模块重试 When 查看模块行 Then 日期和执行时间更新为模块重试对应的完整执行结果。

### F5 通过率详情弹窗

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 点击模块通过率后展示当前模块所有测试用例。
  - 弹窗默认筛选状态为“失败”。
  - 状态筛选只对外提供：失败、通过、跳过。
  - 弹窗顶部筛选参考旧平台截图 1，包含用例名、来源、日期、错误类型、执行状态、查询、重置、Jenkins 任务、测试账号、替换测试账号和更多菜单；其中测试账号、替换测试账号只保留占位入口，本期不实现账号替换逻辑。
  - 弹窗字段：勾选框、用例名、用例简述、错误类型、断言、执行状态、错误信息、确认结果。
  - 失败状态用例支持勾选单个或多个后点击“失败重试”。
  - 弹窗内支持“一键失败重试”，重试当前模块当前筛选范围内所有失败用例；默认筛选失败时等价于重试本模块全部当前失败用例。
  - 只有管理人员可手动修改用例状态；普通成员只能查看状态和触发允许的重试操作。
- **关联数据表**：`test_case_result`
- **验收标准**：
  - `AC5.1` — Given 模块列表存在某模块 When 点击该模块通过率 Then 打开用例详情弹窗。
  - `AC5.2` — Given 弹窗首次打开 When 未手动修改筛选 Then 默认只展示失败状态用例。
  - `AC5.3` — Given 用户选择“通过”筛选 When 查询 Then 只展示通过状态用例。
  - `AC5.4` — Given 用户选择“跳过”筛选 When 查询 Then 只展示跳过状态用例。
  - `AC5.5` — Given 错误信息包含长堆栈 When 页面展示 Then 表格只展示摘要，详情查看需受权限控制并避免泄露敏感凭据。
  - `AC5.6` — Given 详情弹窗中勾选 1 条或多条失败用例 When 点击“失败重试” Then 只为已勾选用例创建失败重试 Jenkins 任务。
  - `AC5.7` — Given 详情弹窗存在失败用例 When 点击弹窗内“一键失败重试” Then 为当前模块全部当前失败用例创建失败重试 Jenkins 任务。
  - `AC5.8` — Given 普通成员打开详情弹窗 When 尝试修改用例状态 Then 页面不展示状态修改入口或后端返回 403。
  - `AC5.9` — Given 管理人员修改用例状态 When 提交成功 Then 当前用例状态、模块失败数和通过率按新状态重新计算。

### F6 一键失败重试

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 在模块行右侧点击“一键失败重试”，重试本模块当前全部失败用例。
  - 模块行右侧不提供单条失败重试；单条或勾选失败重试只能在通过率详情弹窗内操作。
  - 平台通过 Jenkins 失败重试 Job 执行，不直接执行 pytest。
  - 失败重试不更新模块“日期”和“执行时间”。
  - 失败重试完成后，更新失败用例状态、失败数、通过率和 Jenkins 任务记录。
  - 同一测试环境同一模块同时只能存在一个 `queued`、`running` 或 `canceling` 状态的 Jenkins 任务；已有执行中任务时再次重试提示“已有用例重试，无法执行！”。
- **关联数据表**：`jenkins_task`、`module_execution_lock`、`module_snapshot`、`test_case_result`
- **验收标准**：
  - `AC6.1` — Given 模块存在失败用例 When 点击一键失败重试 Then 创建失败重试 Jenkins 任务。
  - `AC6.2` — Given 模块通过率为 100% When 点击一键失败重试 Then 不触发 Jenkins，并提示“通过率 100% 无需失败重试”。
  - `AC6.3` — Given 失败重试执行成功且部分失败用例通过 When 同步结果 Then 对应用例状态变为通过，模块失败数下降，通过率更新。
  - `AC6.4` — Given 失败重试完成 When 查看模块行 Then 日期和执行时间不变。
  - `AC6.5` — Given 同一环境同一模块已有执行中任务 When 再次点击一键失败重试 Then 返回提示“已有用例重试，无法执行！”。

### F7 模块重试

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 在模块行点击“模块重试”，重跑当前模块的全部用例。
  - 平台通过 Jenkins 模块重试 Job 执行，不直接执行 pytest。
  - 模块重试完成后更新模块“日期”和“执行时间”。
  - 模块重试会以本次完整执行结果重置模块快照和当前用例结果。
- **关联数据表**：`jenkins_task`、`module_execution_lock`、`module_snapshot`、`test_case_result`、`module_run_history`
- **验收标准**：
  - `AC7.1` — Given 用户点击模块重试 When 模块无执行锁 Then 创建模块重试 Jenkins 任务。
  - `AC7.2` — Given 模块重试成功完成 When DRF 同步结果 Then 模块快照统计、日期和执行时间全部更新。
  - `AC7.3` — Given 模块重试产生新的失败用例 When 同步结果 Then 旧的当前用例结果归档，新结果成为当前展示数据。
  - `AC7.4` — Given 同一模块已有失败重试运行中 When 点击模块重试 Then 返回提示“已有用例重试，无法执行！”。

### F8 近 7 天、近 30 天模块趋势数据

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 模块行提供“近 7 天测试数据”和“近 30 天测试数据”入口。
  - 7 天和 30 天数据必须按模块隔离，点击某个模块尾部入口后只展示该模块对应窗口内的数据。
  - 使用独立弹窗展示，弹窗标题包含模块名和统计窗口。
  - 弹窗包含通过率表格数据和折线图变化趋势；折线图至少展示通过率随日期变化，表格至少展示日期、总用例数、失败数、通过率、执行时间、运行类型。
  - 趋势数据来自历史模块执行记录，不由前端临时计算。
- **关联数据表**：`module_run_history`
- **验收标准**：
  - `AC8.1` — Given 模块存在近 7 天历史 When 点击该模块“近 7 天测试数据” Then 打开独立弹窗并展示该模块 7 天内通过率表格和折线趋势。
  - `AC8.2` — Given 模块存在近 30 天历史 When 点击该模块“近 30 天测试数据” Then 打开独立弹窗并展示该模块 30 天内通过率表格和折线趋势。
  - `AC8.3` — Given 模块 A 和模块 B 都有历史数据 When 打开模块 A 趋势弹窗 Then 不展示模块 B 的历史数据。
  - `AC8.4` — Given 模块没有对应窗口历史数据 When 打开趋势弹窗 Then 展示空态，不报前端错误。

### F9 Jenkins 任务弹窗

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 模块行提供“Jenkins任务”按钮，点击后打开当前模块今天的 Jenkins 执行情况弹窗。
  - 弹窗字段：`id(job_id)`、任务名、测试环境 URL、状态（Jenkins job 执行状态）、操作。
  - 本期操作只实现“取消任务”“查看报告”“查看Jenkins任务”；截图中其它操作暂不实现。
  - “取消任务”只允许取消当前模块当前环境下 `queued` 或 `running` 的 Jenkins job；取消后释放模块执行锁。
  - “查看报告”直接跳转对应 Allure 报告链接。
  - “查看Jenkins任务”直接跳转本条 Jenkins job 页面。
  - 一个测试环境下的一个模块，同时仅能有一条正在执行的 job；若已有正在执行 job，重试类操作统一提示“已有用例重试，无法执行！”。
- **关联数据表**：`jenkins_task`、`module_execution_lock`
- **验收标准**：
  - `AC9.1` — Given 模块今天存在 Jenkins 任务 When 点击“Jenkins任务” Then 弹窗展示 job_id、任务名、测试环境 URL、状态和操作。
  - `AC9.2` — Given Jenkins 任务状态为 `queued` 或 `running` When 点击“取消任务” Then 后端调用 Jenkins 取消接口并将任务状态流转为 `canceling`，最终同步为 `canceled`。
  - `AC9.3` — Given Jenkins 任务已生成 Allure 报告链接 When 点击“查看报告” Then 浏览器跳转到该 Allure 报告链接。
  - `AC9.4` — Given Jenkins 任务已生成 build 链接 When 点击“查看Jenkins任务” Then 浏览器跳转到该 Jenkins job 页面。
  - `AC9.5` — Given 同一环境同一模块存在 `queued`、`running` 或 `canceling` 任务 When 用户点击失败重试或模块重试 Then 不创建新任务并提示“已有用例重试，无法执行！”。
  - `AC9.6` — Given 用户取消任务成功 When 再次触发失败重试或模块重试 Then 可以创建新的 Jenkins 任务。

### F10 用户管理与邀请码注册

- **能做什么 / 做到什么程度 / 满足什么要求**：
  - 系统提供用户登录、邀请码注册和基础用户管理能力，作为用例状态手动修改权限的前置能力。
  - 用户角色仅有管理人员和普通成员。
  - 管理人员可生成邀请码、查看邀请码状态、作废未使用的邀请码，并分享给待注册成员。
  - 普通成员只能使用有效邀请码注册，不允许无邀请码自由注册。
  - 一个邀请码只能使用一次，成功注册后绑定使用人和使用时间。
  - 只有管理人员可手动修改用例状态；所有手动修改需记录操作人、原状态、新状态和原因。
- **关联数据表**：`user_account`、`invitation_code`、`case_status_audit`
- **验收标准**：
  - `AC10.1` — Given 管理人员登录后台 When 生成邀请码 Then 系统创建一条未使用的邀请码并展示过期时间。
  - `AC10.2` — Given 普通成员使用有效邀请码注册 When 注册成功 Then 邀请码状态变为已使用且绑定该用户。
  - `AC10.3` — Given 邀请码已使用、过期或作废 When 再次注册 Then 后端拒绝并返回明确错误。
  - `AC10.4` — Given 普通成员访问用例状态修改接口 When 提交修改 Then 返回 403。
  - `AC10.5` — Given 管理人员修改用例状态 When 提交原因并保存 Then 生成审计记录，并同步更新模块统计。

## §5 状态机定义 [必填]

### 实体：`test_case_result.display_status`

- **状态全集**：`failed`（失败）、`passed`（通过）、`skipped`（跳过）、`archived`（归档）

| 源状态 | 事件 / 操作 | 目标状态 | 守卫条件 | 副作用 |
| --- | --- | --- | --- | --- |
| `failed` | 失败重试通过 | `passed` | Jenkins 返回该用例通过 | 模块失败数下降，通过率更新；不更新日期和执行时间 |
| `failed` | 管理人员标记跳过 | `skipped` | 用户角色为管理人员且填写修改原因 | 模块失败数下降，通过率更新；写入状态审计 |
| `skipped` | 管理人员恢复失败 | `failed` | 用户角色为管理人员且填写修改原因 | 模块失败数上升，通过率更新；写入状态审计 |
| `failed/passed/skipped` | 管理人员手动改为目标状态 | `failed/passed/skipped` | 目标状态合法、原因非空、不能修改已归档结果 | 重新计算模块统计；写入状态审计 |
| `failed/skipped/passed` | 模块重试开始归档旧结果 | `archived` | 模块重试任务进入同步阶段 | 旧结果不再作为当前详情展示 |
| 任意 | 每日全量或模块重试写入新结果 | `failed/passed/skipped` | 根据 pytest/Jenkins 结果解析 | 新结果成为当前展示数据 |

### 实体：`jenkins_task.status`

- **状态全集**：`queued`、`running`、`success`、`failed`、`canceling`、`canceled`

| 源状态 | 事件 / 操作 | 目标状态 | 守卫条件 | 副作用 |
| --- | --- | --- | --- | --- |
| 无 | 创建 Jenkins 任务 | `queued` | 参数合法且模块锁获取成功 | 写入任务记录 |
| `queued` | Jenkins 分配 build | `running` | 获取 build number | 记录 build URL |
| `running` | Jenkins 成功完成 | `success` | 构建结果 SUCCESS | 同步报告与快照，释放模块锁 |
| `running` | Jenkins 失败完成 | `failed` | 构建结果 FAILURE/UNSTABLE 或同步失败 | 记录失败摘要，释放模块锁 |
| `queued/running` | 用户取消 | `canceling` | 用户为管理人员或任务触发人，且任务属于当前环境模块 | 调用 Jenkins 取消 |
| `canceling` | Jenkins 确认取消 | `canceled` | Jenkins 返回取消结果 | 释放模块锁 |

## §7 API 契约 [必填 · 冻结后前后端共同依据]

> API 使用 `/api/v1` 前缀；响应采用 `{ data, meta, links }` 或 `{ error }` 包装；列表接口必须后端分页、筛选和排序，避免复刻旧平台一次性拉取全部数据后前端过滤。

### `POST /api/v1/auth/login`

- **用途 / 权限**：用户登录；公开接口。
- **请求体**：

| 字段 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `username` | string | 是 | 长度 1-64 | 登录名 |
| `password` | string | 是 | 长度 8-128 | 密码 |

- **成功响应**：`200 OK`，返回访问令牌、用户基础信息和角色。

### `POST /api/v1/auth/register`

- **用途 / 权限**：邀请码注册；公开接口但必须校验邀请码。
- **请求体**：

| 字段 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `invitation_code` | string | 是 | 未使用、未过期、未作废 | 一次性邀请码明文，仅请求时提交 |
| `username` | string | 是 | 长度 1-64，唯一 | 登录名 |
| `display_name` | string | 是 | 长度 1-128 | 展示名 |
| `password` | string | 是 | 长度 8-128 | 密码 |

- **成功响应**：`201 Created`，创建普通成员或邀请码指定角色用户，并将邀请码原子更新为 `used`。
- **错误码**：

| HTTP | 业务码 | 含义 | 触发条件 |
| --- | --- | --- | --- |
| 409 | `username_exists` | 用户名已存在 | 用户名重复 |
| 422 | `invalid_invitation_code` | 邀请码不可用 | 已使用、已过期、已作废或不存在 |

### `GET /api/v1/users`

- **用途 / 权限**：用户列表；仅管理人员可读。
- **请求参数**：`role`、`is_active`、`page`、`per_page`。
- **成功响应**：`200 OK`，返回用户 id、用户名、展示名、角色、启用状态和最近登录时间。

### `POST /api/v1/invitations`

- **用途 / 权限**：生成邀请码；仅管理人员可执行。
- **请求体**：

| 字段 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `role_to_grant` | string | 否 | `admin` / `member` | 注册后授予角色，默认 `member` |
| `expires_at` | datetime | 是 | 晚于当前时间 | 过期时间 |

- **成功响应**：`201 Created`，仅本次响应返回邀请码明文，数据库保存哈希值。

### `GET /api/v1/invitations`

- **用途 / 权限**：查看邀请码列表；仅管理人员可读。
- **请求参数**：`status`、`page`、`per_page`。
- **成功响应**：`200 OK`，返回邀请码状态、授权角色、创建人、使用人、过期时间和使用时间，不返回明文邀请码。

### `POST /api/v1/invitations/{invitation_id}/revoke`

- **用途 / 权限**：作废未使用邀请码；仅管理人员可执行。
- **成功响应**：`200 OK`，返回作废后的邀请码记录。
- **错误码**：`409 invitation_already_used` 表示邀请码已使用不能作废。

### `GET /api/v1/test-environments`

- **用途 / 权限**：获取可用测试环境列表；登录用户可读。
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `is_active` | query | boolean | 否 | true/false | 是否只返回启用环境 |

- **成功响应**：`200 OK`

```json
{
  "data": [
    {
      "id": 1,
      "env_key": "mock-gbif",
      "env_name": "模拟测试环境",
      "base_url": "https://api.gbif.org"
    }
  ]
}
```

### `GET /api/v1/test-environments/{environment_id}/summary`

- **用途 / 权限**：获取环境通过率页汇总；登录用户可读。
- **请求参数**：`environment_id` path 必填。
- **成功响应**：`200 OK`，返回环境、开始时间、结束时间、通过率、运行时间和操作可用性。
- **错误码**：

| HTTP | 业务码 | 含义 | 触发条件 |
| --- | --- | --- | --- |
| 404 | `environment_not_found` | 环境不存在 | `environment_id` 无效 |

### `GET /api/v1/module-snapshots`

### `GET /api/v1/module-snapshots/{snapshot_id}/cases`

- **用途 / 权限**：获取模块用例详情；登录用户可读。默认筛选失败。
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `snapshot_id` | path | integer | 是 | 存在 | 模块快照 |
| `status` | query | string | 否 | `failed` / `passed` / `skipped` | 默认 `failed` |
| `case_name` | query | string | 否 | 长度 <= 256 | 用例名模糊筛选 |
| `node_id` | query | string | 否 | 长度 <= 1024 | 用例来源 / pytest node id |
| `error_type` | query | string | 否 | 多值逗号分隔 | 错误类型 |
| `page` / `per_page` | query | integer | 否 | 分页 | 分页参数 |

- **成功响应**：`200 OK`，返回用例结果 ID、用例名、来源、用例简述、错误类型、断言、执行状态、错误信息摘要、确认结果和是否可勾选重试。

### `POST /api/v1/module-snapshots/{snapshot_id}/failed-case-retries`

- **用途 / 权限**：失败用例重试；登录用户可执行，后续如需限制可收敛为管理人员。本接口同时支撑模块行一键失败重试、详情弹窗勾选失败重试和详情弹窗一键失败重试。
- **请求参数**：`snapshot_id` path 必填。
- **请求体**：

| 字段 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `retry_scope` | string | 是 | `all_failed` / `selected_failed` | `all_failed` 表示当前模块全部失败用例；`selected_failed` 表示只重试勾选用例 |
| `case_result_ids` | array[integer] | 否 | `selected_failed` 时必填，且均属于当前模块当前失败用例 | 勾选的失败用例结果 ID |

- **成功响应**：`202 Accepted`，返回创建的 Jenkins 任务。
- **错误码**：

| HTTP | 业务码 | 含义 | 触发条件 |
| --- | --- | --- | --- |
| 409 | `module_execution_locked` | 已有用例重试，无法执行！ | 同环境同模块已有有效锁 |
| 422 | `invalid_case_selection` | 勾选用例不可重试 | 勾选为空、跨模块、非当前结果或非失败状态 |
| 422 | `no_failed_cases` | 无需失败重试 | 当前失败数为 0 |
| 503 | `jenkins_unavailable` | Jenkins 不可用 | Jenkins API 调用失败 |

### `POST /api/v1/module-snapshots/{snapshot_id}/module-reruns`

- **用途 / 权限**：模块重试；登录用户可执行，后续如需限制可收敛为管理人员。
- **请求参数**：`snapshot_id` path 必填。
- **成功响应**：`202 Accepted`，返回创建的 Jenkins 任务。
- **错误码**：同失败重试；当同环境同模块已有有效锁时返回 `409 module_execution_locked`，错误消息固定为“已有用例重试，无法执行！”。

### `GET /api/v1/module-snapshots/{snapshot_id}/trend`

- **用途 / 权限**：获取近 7 天或近 30 天模块测试数据；登录用户可读。
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `days` | query | integer | 是 | 7 或 30 | 统计窗口 |

- **成功响应**：`200 OK`，返回日期、通过率、失败数、执行时间序列。

### `PATCH /api/v1/case-results/{case_result_id}/status`

- **用途 / 权限**：管理人员手动修改用例状态；仅管理人员可执行。
- **请求体**：

| 字段 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `display_status` | string | 是 | `failed` / `passed` / `skipped` | 目标状态 |
| `reason` | string | 是 | 长度 1-512 | 修改原因 |

- **成功响应**：`200 OK`，返回更新后的用例结果、模块统计摘要和审计记录 ID。
- **错误码**：

| HTTP | 业务码 | 含义 | 触发条件 |
| --- | --- | --- | --- |
| 403 | `admin_required` | 仅管理人员可修改用例状态 | 普通成员调用 |
| 409 | `archived_case_result` | 已归档结果不可修改 | `is_current=false` 或状态为 `archived` |

### `GET /api/v1/module-snapshots/{snapshot_id}/jenkins-tasks`

- **用途 / 权限**：获取当前模块今天的 Jenkins 任务弹窗数据；登录用户可读。

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |

| `status` | query | string | 否 | 任务状态枚举 | 可按 Jenkins 状态筛选 |
| `page` / `per_page` | query | integer | 否 | 分页 | 分页参数 |

- **成功响应**：`200 OK`

    {
      "id": 301,

      "job_name": "api-test-module-rerun",

### `POST /api/v1/jenkins-tasks/{task_id}/cancel`

- **用途 / 权限**：取消当前执行中的 Jenkins job；管理人员或任务触发人可执行。
- **成功响应**：`202 Accepted`，返回任务当前状态 `canceling`。

### 弹窗：模块用例详情

| 元素 | 字段来源（对应 §7 API 字段） | 状态枚举 / 标签 | 加载/空/错误/权限态 | 操作与反馈 |

| 手动改状态 | `PATCH /case-results/{id}/status` | 失败 / 通过 / 跳过 | 仅管理人员可见；原因必填 | 修改后刷新统计并写审计 |
| 测试账号 / 替换测试账号 | 本期占位 | 后续功能 | 登录用户可见但不可提交真实替换动作 | 点击提示后续实现 |

### 弹窗：模块趋势数据

## §10 容器化兼容检查 [必填 · 质量门禁]

| 检查项 | 是否存在 | 整改方案 |

## §11 非功能要求 [S可简]

- **性能**：模块列表和用例详情必须后端分页；默认每页 20 条，最大 100 条。错误详情长文本按需展开或懒加载。
- **安全**：错误信息展示前做脱敏，避免 token、cookie、租户密钥、真实账号和敏感 URL 直接暴露；邀请码不保存明文；手动改状态必须记录审计。
- **可用性**：Jenkins 不可用时，读页面不受影响；触发类操作返回明确错误。
- **一致性**：失败重试只更新统计和用例状态，不更新模块日期和执行时间；模块重试和每日全量才更新完整执行基线；同一环境同一模块同一时刻只能存在一个执行中 Jenkins job。
- **可观测性**：所有 Jenkins 触发、同步失败、状态变更和用户操作需写入任务记录或审计日志。

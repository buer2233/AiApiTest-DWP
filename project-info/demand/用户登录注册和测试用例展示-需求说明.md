## §1 需求背景与目标

- **背景**：项目当前已完成平台架构、固定开发 loop、Jenkins 主干和 Docker 化约束设计，但缺少可维护的首个前后端业务闭环源码。首期需要先具备用户进入平台的身份入口，并能展示仓库中已有接口自动化测试用例，为后续模块通过率、失败用例、Jenkins 任务和报告入口打基础。
- **目标**：
  - 访问者可以通过邀请码注册平台账号。
  - 管理员可以创建、查看和禁用注册邀请码。
  - 用户可以通过登录页登录平台，登录后进入测试用例展示页面。
  - 已登录用户可以查看平台同步出的测试用例列表。
  - 未登录用户不能访问测试用例页面或测试用例 API。
  - 平台后端具备可测试、可迁移的用户和测试用例数据模型。
- **成功指标 / 价值**：
  - 后端 pytest 接口测试覆盖注册、登录、登出/当前用户、用例同步/列表和权限拒绝。
  - 前端 Playwright 覆盖登录、注册和测试用例展示主流程。
  - 首期需求产物可作为后续功能的基线，不依赖 `.pyc`、本机绝对路径或真实凭据。

## §2 范围

- **做（in scope）**：
  - 邀请码注册 API 与注册页面。
  - 邀请码管理 API 与管理页面，首期支持创建、查看、禁用，不做编辑和删除。
  - 用户登录 API、登录页面、当前用户 API 和前端登录态保护。
  - 测试用例元数据表设计。
  - 从 `api-test/test_case/` 扫描并同步测试用例元数据的后端服务。
  - 测试用例列表 API，支持分页、关键字筛选和模块/包名筛选。
  - 测试用例展示页面，含筛选、表格、分页、加载态、空状态、错误态和未登录跳转。
- **不做（out of scope）**：
  - 不触发 Jenkins 任务，不执行 pytest，不生成 Allure 报告。
  - 不实现模块重试、失败重试、失败用例明细和报告入口。
  - 不实现完整用户管理、审批列表、密码找回、多租户、验证码、短信、邮箱激活、邀请码批量导入、邀请码邮件发送、邀请码删除或邀请码编辑。
  - 不接入真实生产环境、真实 Jenkins 凭据或真实业务地址。

## §4 功能清单与验收标准

### F1 用户注册

- **能做什么 / 做到什么程度 / 满足什么要求**：访问者可以提交邀请码、用户名、邮箱、密码和确认密码创建平台账号。系统校验邀请码、必填项、邮箱格式、用户名唯一、邮箱唯一和两次密码一致。注册成功后的用户状态为 `active`，可以立即登录。
- **关联数据表**：`accounts_user`
- **验收标准（Given-When-Then）**：
  - `AC1.1` — Given 未登录访问者 When 使用有效邀请码、合法用户名、邮箱和一致密码提交注册 Then 系统创建 `active` 用户并返回注册成功信息。
  - `AC1.2` — Given 未登录访问者 When 使用已存在用户名或邮箱提交注册 Then 系统拒绝创建并返回字段级错误。
  - `AC1.3` — Given 未登录访问者 When 两次密码不一致或密码长度不足 Then 系统拒绝创建并提示具体原因。
  - `AC1.4` — Given 未登录访问者 When 使用无效邀请码提交注册 Then 系统拒绝创建用户并返回邀请码错误。
  - `AC1.5` — Given 注册成功用户 When 使用注册账号登录 Then 系统允许登录。
- **异常场景**：
  - 请求体缺少必填字段 → `400 validation_error`。
  - 用户名或邮箱重复 → `409 duplicate_user` 或字段级唯一性错误。
  - 密码不符合规则 → `400 validation_error`。
  - 邀请码缺失或错误 → `400 invalid_invite_code`。
- **边界值**：
  - 用户名最短 3 个字符、最长 30 个字符。
  - 密码最短 8 个字符。
  - 邮箱最长按 Django 默认邮箱字段限制。
- **并发 / 幂等**：两个并发请求注册同一用户名或邮箱时，只允许一个成功，另一个返回唯一性冲突。

### F2 用户登录、退出和当前用户

- **能做什么 / 做到什么程度 / 满足什么要求**：用户可以使用用户名或邮箱加密码登录。登录成功后前端保存认证凭据并访问受保护页面。用户可以获取当前用户信息并退出登录。
- **关联数据表**：`accounts_user`，`authtoken_token`
- **验收标准（Given-When-Then）**：
  - `AC2.1` — Given 已注册且允许登录的用户 When 提交正确账号和密码 Then 系统返回认证凭据和用户基础信息。
  - `AC2.2` — Given 已注册用户 When 提交错误密码 Then 系统拒绝登录并返回统一错误，不泄露账号是否存在的敏感细节。
  - `AC2.3` — Given 未登录用户 When 访问当前用户 API 或测试用例列表 API Then 系统返回 `401 unauthorized`。
  - `AC2.4` — Given 已登录用户 When 请求当前用户 API Then 系统返回用户 id、用户名、邮箱、角色和状态。
  - `AC2.5` — Given 已登录用户 When 执行退出登录 Then 当前认证凭据失效或前端清除登录态，随后访问受保护资源需要重新登录。
- **异常场景**：
  - 账号或密码为空 → `400 validation_error`。
  - 账号不存在或密码错误 → `401 invalid_credentials`。
  - 用户状态不允许登录（如 Q1 选审批） → `403 user_not_active`。
- **边界值**：
  - 登录账号支持用户名或邮箱，但不得支持空白字符串。
- **并发 / 幂等**：重复登录应返回可用凭据；重复退出不应造成服务端错误。

### F3 邀请码管理

- **能做什么 / 做到什么程度 / 满足什么要求**：admin 可以创建、查看和禁用邀请码。邀请码用于注册校验，支持最大使用次数和过期时间。member 和未登录用户不能管理邀请码。首期不支持编辑、删除、批量导入或邮件发送。
- **关联数据表**：`registration_invite_code`
- **验收标准（Given-When-Then）**：
  - `AC3.1` — Given admin 已登录 When 创建邀请码并填写最大使用次数和可选过期时间 Then 系统创建可用邀请码并返回邀请码详情。
  - `AC3.2` — Given member 已登录 When 调用创建邀请码 API Then 系统返回 `403 forbidden`。
  - `AC3.3` — Given admin 已登录且存在邀请码 When 查询邀请码列表 Then 系统分页返回邀请码、状态、最大使用次数、已使用次数、过期时间和创建人。
  - `AC3.4` — Given admin 已登录 When 禁用邀请码 Then 该邀请码状态变为 `disabled`，后续注册不可使用。
  - `AC3.5` — Given 邀请码已达到最大使用次数或已过期 When 访问者用该邀请码注册 Then 系统拒绝注册并返回邀请码不可用。
- **异常场景**：
  - 创建重复邀请码 → `409 duplicate_invite_code`。
  - 最大使用次数小于 1 → `400 validation_error`。
  - 过期时间早于当前时间 → `400 validation_error`。
  - 非 admin 调用管理接口 → `403 forbidden`。
- **边界值**：
  - 邀请码长度 6-64 个字符。
  - 最大使用次数最小为 1。
  - 过期时间可为空；为空表示不过期。
- **并发 / 幂等**：两个注册请求并发消耗同一个剩余次数为 1 的邀请码时，只允许一个注册成功，另一个返回邀请码不可用。

### F4 测试用例元数据同步

- **能做什么 / 做到什么程度 / 满足什么要求**：后端从 `api-test/test_case/` 下解析 pytest 测试文件，提取测试用例模块、文件路径、类名、函数名、node id 和可读描述，同步到数据库供列表展示。同步入口仅 admin 可调用，member 只能查看列表。
- **关联数据表**：`test_case_item`
- **验收标准（Given-When-Then）**：
  - `AC4.1` — Given 仓库存在 `api-test/test_case/` When 后端执行测试用例同步 Then 系统将可识别的 pytest 用例写入 `test_case_item`。
  - `AC4.2` — Given 同一测试用例被重复同步 When node id 未变化 Then 系统更新元数据而不是重复创建。
  - `AC4.3` — Given 测试文件中存在函数 docstring 或 allure title When 同步 Then 系统优先使用可读标题/描述展示。
  - `AC4.4` — Given 扫描路径不存在或没有用例 When 执行同步 Then 系统返回明确错误或空结果，不产生未处理异常。
- **异常场景**：
  - `api-test/test_case/` 不存在 → `500 sync_source_missing` 或同步命令失败并记录错误。
  - 解析单个文件失败 → 跳过该文件并记录错误，不能中断所有可解析用例同步。
- **边界值**：
  - 空目录同步结果为 0 条。
  - 单个 node id 长度按 512 字符上限保存。
- **并发 / 幂等**：同步按 `node_id` 唯一更新，重复执行结果稳定。

### F5 测试用例展示

- **能做什么 / 做到什么程度 / 满足什么要求**：已登录用户可以在前端查看测试用例列表，支持分页、关键字搜索和模块/包名筛选。页面展示加载、空状态、错误状态和未登录跳转。
- **关联数据表**：`test_case_item`
- **验收标准（Given-When-Then）**：
  - `AC5.1` — Given 已登录用户且数据库存在测试用例 When 访问测试用例页面 Then 页面展示测试用例表格和总数。
  - `AC5.2` — Given 已登录用户 When 输入关键字搜索 Then 页面只展示 node id、函数名或描述匹配的测试用例。
  - `AC5.3` — Given 已登录用户 When 按模块/用例包筛选 Then 页面只展示对应模块/包下的测试用例。
  - `AC5.4` — Given 测试用例列表为空 When 访问页面 Then 页面展示空状态，不显示错误。
  - `AC5.5` — Given API 请求失败 When 页面加载测试用例 Then 页面展示可恢复错误状态并允许重试。
  - `AC5.6` — Given 未登录用户 When 访问测试用例页面 Then 前端跳转到登录页。
- **异常场景**：
  - 分页参数非法 → 后端使用默认分页或返回 `400 validation_error`。
  - 认证凭据失效 → 前端清除登录态并跳转登录页。
- **边界值**：
  - 默认每页 20 条，允许 `page_size` 10、20、50。
  - 搜索关键字前后空白应被忽略。
- **并发 / 幂等**：列表查询为只读操作，重复请求不改变数据。

## §5 状态机定义

- **实体**：`accounts_user.status`
- **状态全集**：`active`、`pending`、`disabled`
- **迁移表**：

| 源状态 | 事件 / 操作 | 目标状态 | 守卫条件 | 副作用 |
| --- | --- | --- | --- | --- |
| 无 | 注册成功 | `active` | 邀请码有效且用户字段校验通过 | 创建用户并允许登录 |
| `active` | 管理员禁用（本期不做页面） | `disabled` | 后续需求实现 | 禁止登录 |
| `pending` | 管理员审批（本期不实现） | `active` | 后续需求实现 | 允许登录 |

- **实体**：`test_case_item.sync_status`
- **状态全集**：`synced`、`missing`
- **迁移表**：

| 源状态 | 事件 / 操作 | 目标状态 | 守卫条件 | 副作用 |
| --- | --- | --- | --- | --- |
| 无 | 首次同步发现用例 | `synced` | node id 可解析 | 新增用例 |
| `synced` | 再次同步仍发现用例 | `synced` | node id 不变 | 更新路径、描述、模块等元数据 |
| `synced` | 再次同步未发现用例 | `missing` | 保留历史记录 | 列表默认不展示或以筛选方式展示，最终策略待后续扩展 |

- **实体**：`registration_invite_code.status`
- **状态全集**：`active`、`disabled`、`exhausted`、`expired`
- **迁移表**：

| 源状态 | 事件 / 操作 | 目标状态 | 守卫条件 | 副作用 |
| --- | --- | --- | --- | --- |
| 无 | admin 创建邀请码 | `active` | code 唯一，max_uses >= 1，expires_at 为空或晚于当前时间 | 邀请码可用于注册 |
| `active` | 注册成功消耗邀请码 | `active` 或 `exhausted` | used_count + 1 小于 max_uses 保持 active；达到 max_uses 变 exhausted | 累加 used_count，记录最近使用时间 |
| `active` | 到达过期时间 | `expired` | 当前时间大于 expires_at | 后续注册不可使用 |
| `active` | admin 禁用 | `disabled` | admin 权限 | 后续注册不可使用 |

## §6 数据表设计

### 表 `accounts_user`

- **用途**：保存平台用户、登录身份、角色和账号状态。
- **写入策略**：注册新增；登录只读；后续用户管理需求可更新角色和状态。
- **关键约束**：`username` 唯一，`email` 唯一。

| 字段 | 类型建议 | 必填 | 默认 | 说明 | 索引 / 约束 |
| --- | --- | --- | --- | --- | --- |
| `id` | bigint pk | 是 |  | 主键 | PK |
| `username` | varchar(30) | 是 |  | 登录用户名 | unique |
| `email` | varchar(254) | 是 |  | 邮箱 | unique |
| `password` | varchar(128) | 是 |  | Django 哈希密码 |  |
| `role` | varchar(20) | 是 | `member` | `admin` / `member` | index |
| `status` | varchar(20) | 是 | `active` | `active` / `pending` / `disabled` | index |
| `is_staff` | bool | 是 | false | Django 管理后台标识 |  |
| `is_superuser` | bool | 是 | false | Django 超级用户标识 |  |

### POST `/api/v1/auth/register`

- **用途 / 权限**：开放注册平台账号；未登录可访问。
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `username` | body | string | 是 | 3-30 位，字母数字下划线优先 | 用户名 |
| `email` | body | string | 是 | 邮箱格式，唯一 | 邮箱 |
| `password` | body | string | 是 | 最短 8 位 | 密码 |
| `password_confirm` | body | string | 是 | 必须等于 `password` | 确认密码 |
| `invite_code` | body | string | 是 | 必须匹配配置的邀请码 | 注册邀请码 |

- **成功响应**：`201 Created`

```json
{
  "data": {
    "id": 1,
    "username": "demo_user",
    "email": "demo@example.com",
    "role": "member",
    "status": "active"
  }
}
```

- **错误码**：

| HTTP | 业务码 | 含义 | 触发条件 |
| --- | --- | --- | --- |
| 400 | `validation_error` | 请求参数不合法 | 缺字段、邮箱格式错误、密码不一致 |
| 400 | `invalid_invite_code` | 邀请码错误 | 邀请码缺失或不匹配 |
| 400 | `invite_code_unavailable` | 邀请码不可用 | 邀请码已禁用、已用尽或已过期 |
| 409 | `duplicate_user` | 用户名或邮箱已存在 | 唯一约束冲突 |

- **分页 / 筛选 / 排序**：无。
- **关键状态流转 / 幂等**：注册不是幂等操作；重复用户名/邮箱返回冲突。

- **用途 / 权限**：用户登录；未登录可访问。
- **请求参数**：

| 参数 | 位置 | 类型 | 必填 | 校验 | 说明 |

| `account` | body | string | 是 | 非空 | 用户名或邮箱 |
| `password` | body | string | 是 | 非空 | 密码 |

- **成功响应**：`200 OK`，返回 DRF Token 凭据：

```json

    "token": "example-token-placeholder",

      "username": "demo_user",
```

| 401 | `invalid_credentials` | 凭据错误 | 账号不存在或密码错误 |
| 403 | `user_not_active` | 用户状态不允许登录 | pending 或 disabled |

- **关键状态流转 / 幂等**：登录可重复执行，返回当前认证凭据。

## §10 容器化兼容检查

| 检查项 | 是否存在 | 整改方案 |

| 本机绝对路径 | 否 | 测试用例扫描路径使用仓库相对路径 `api-test/test_case`，由后端配置计算。 |

## §11 非功能要求

- **性能**：测试用例列表默认分页 20 条；关键字搜索首期可使用数据库 `icontains`，后续大规模数据再引入索引优化。
- **安全**：密码必须哈希存储；登录错误不泄露账号存在性；注册必须校验邀请码；邀请码管理仅 admin 可访问；受保护 API 必须认证。
- **可用性**：前端表单和列表必须具备加载、错误、空状态和可恢复操作。
- **兼容性**：后端测试环境可使用 SQLite，开发/验收环境使用 MySQL；通过环境变量切换。
- **可观测性**：同步测试用例时返回新增、更新、缺失和总数，便于验收。

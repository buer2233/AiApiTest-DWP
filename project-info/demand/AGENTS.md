# project-info/demand/AGENTS.md

本目录是固定 loop 第 1 阶段：需求分析阶段。进入本目录工作前，必须先遵守根目录 `AGENTS.md` 和 `project-info/AGENTS.md`，再遵守本文件。

## 阶段目标

- 输出详细需求说明书，也就是概要设计。
- 明确需求背景、目标、范围、不做事项、用户角色、业务流程、功能清单、非功能要求、验收标准和变更记录。
- 同步完成功能对应的数据表设计，包括字段、关联关系、更新规则、状态流转和关键约束。

## 技能要求

- 编写或修改需求分析资料时，优先使用 `product-requirements`。
- 新需求、新功能、架构或行为变更前，按根规则使用 `brainstorming` 做方案确认。
- 如果技能不可用，应记录原因，并在不跳过需求分析质量要求的前提下继续补齐文档。

## 文件命名

- 同一需求后续阶段必须沿用同一需求名，仅后缀不同。
- 推荐格式：`需求名-需求说明.md`。
- 草稿可使用 `需求草稿-主题.md`，但进入正式开发前应整理成正式需求说明书。

## 内容要求

- 每个功能必须说明能做什么、做到什么程度、满足什么要求。
- 每个功能都要关联对应的数据表或明确不涉及数据持久化。
- 表设计应至少包含表名、用途、字段、字段类型建议、必填性、默认值、索引建议、关联关系和状态流转。
- 明确与 `api-test/`、`jenkins/`、`back-end/`、`front-end/` 的职责边界。
- 验收口径必须能被 `project-info/test_case/` 转换为测试用例。

## 安全要求

- 需求应保持通用测试平台定位，不写入真实账号、密码、token、cookie、租户密钥、生产 URL 或敏感地址。
- 示例数据使用占位符或脱敏数据。

# demand AGENTS.md

本目录用于存放项目整体需求分析资料，包括需求背景、目标范围、用户角色、业务流程、功能清单、非功能要求、验收标准和需求变更记录。

## 强制技能

- 进入本目录编写或修改需求分析资料时，必须使用 `/product-requirements` 技能。
- 该技能来源于 `https://github.com/stellarlinkco/myclaude`。
- 开始工作前必须检查当前环境是否可用该技能。
- 如果检查到该技能无法使用，应推荐用户先安装 `/product-requirements` 技能后再继续，不要直接跳过需求分析技能流程。

## 文件约定

- 需求文档使用简体中文，专业名词可保留英文原名。
- 需求应保持通用测试平台定位，不写入具体公司业务模块常量。
- 推荐文件名：`需求分析.md`、`产品需求说明书.md`、`需求变更记录.md`。

## 内容要求

- 明确目标、范围、不做事项、用户角色、主流程、异常流程和验收标准。
- 涉及功能边界时，要和根目录主计划及各模块 `AGENTS.md` 保持一致。

# project-info/AGENTS.md

本目录是项目说明资料统筹目录，只负责沉淀需求、测试用例、UI 原型、架构图、流程图和交接资料，不存放业务实现代码或运行产物。

## 固定 loop 中的位置

`project-info/` 承载每个需求 loop 的前三个阶段：

- `demand/`：需求分析阶段，输出详细需求说明书和数据库表设计。
- `test_case/`：功能测试用例阶段，输出详细测试用例和验收场景。
- `UI/`：UI 原型图阶段，输出页面原型和交互说明。

除非用户明确说明跳过，否则后端和前端开发前必须能在本目录找到同一需求命名的前置产物。

## 子目录职责

| 目录 | 用途 |
| --- | --- |
| `demand/` | 存放需求背景、目标范围、功能设计、验收口径、数据表设计和变更记录。 |
| `test_case/` | 存放功能测试用例、验收用例、异常场景、边界场景和回归测试清单。 |
| `UI/` | 存放 UI 原型图、页面交互说明、组件布局、页面状态和原型生成提示词。 |
| `project_detail/` | 存放项目架构图、流程图、项目说明图片和架构说明文档。 |

## 命名规则

- 同一需求不同阶段产物命名必须一致，仅后缀不同，并能明确区分阶段。
- 推荐格式：`需求名-需求说明.md`、`需求名-测试用例.md`、`需求名-UI原型.md`。
- 架构阶段资料使用英文文件名，当前架构说明以 `project-info/project_detail/project-architecture.md` 为准。

## 通用规则

- 修改本目录前先读取根目录 `AGENTS.md`、当前需求资料和相关子目录 `AGENTS.md`。
- 资料必须使用简体中文编写，专业名词可保留英文原名。
- 文档应保持通用测试平台定位，不写入不可迁移的业务常量、真实地址或敏感配置。
- 需求、测试用例和 UI 原型之间必须能互相追溯，不能出现只有实现没有设计来源的内容。

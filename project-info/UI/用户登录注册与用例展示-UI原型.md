## 一、设计系统（Design Tokens）

> 所有颜色须落为 Element Plus 主题变量 / 全局 CSS 变量 / 组件 token，页面内**禁止散落硬编码色值**。
> 品牌陶土色仅用于主操作与关键强调，**不替代**状态语义色。

### 1.1 色彩

| 角色 | Token | 参考色值 | 用途 |
| --- | --- | --- | --- |
| 页面背景 | `--bg-base` | `#F7F5F2`（暖米白） | 全局背景 |
| 面板/卡片 | `--bg-surface` | `#FFFFFF` / `#FCFBF9` | 卡片、表格容器 |
| 正文 | `--text-primary` | `#2B2A28`（炭黑） | 标题、主要文字 |
| 次要文字 | `--text-secondary` | `#6B655C`（暖灰） | 辅助说明、占位 |
| 边框 | `--border-base` | `#E8E3DC`（浅暖灰） | 分隔线、输入框边框 |
| **品牌主色** | `--brand-primary` | `#C15F3C`（暖陶土） | 主按钮、链接、关键强调 |
| 品牌悬停 | `--brand-hover` | `#A94E2F` | 主按钮 hover |
| 品牌浅底 | `--brand-soft` | `#F3E4DC` | 选中行/标签浅底 |

**状态语义色（与品牌色严格区分）**

| 状态 | Token | 参考色值 | 用途 |
| --- | --- | --- | --- |
| 成功 | `--status-success` | `#2F855A`（绿） | 成功提示、active 标签 |
| 失败/错误 | `--status-danger` | `#C0392B`（红） | 错误提示、disabled 标签、403/失败 |
| 警告 | `--status-warning` | `#B7791F`（黄） | pending 标签、警示 |
| 信息 | `--status-info` | `#2B6CB0`（蓝） | 一般信息提示 |

**severity 级别标签色**（用例展示页）

| severity | 颜色语义 | 标签样式 |
| --- | --- | --- |
| `blocker` | 最高，深红实心 | danger（加深） |
| `critical` | 高，红 | danger |
| `normal` | 中，蓝灰 | info / 默认 |
| `minor` | 低，灰 | info（浅） |
| `trivial` | 最低，浅灰 | 默认（plain） |
| 空值 | 未标注 | `—` 灰色占位 |

### 1.2 字号层级

| 层级 | 字号 / 字重 | 用途 |
| --- | --- | --- |
| H1 | 24px / 600 | 页面主标题 |
| H2 | 18px / 600 | 区块标题、卡片标题 |
| 正文 | 14px / 400 | 表单标签、表格正文 |
| 辅助 | 12px / 400 | 提示、占位、分页信息 |
| 表格头 | 13px / 600 | 列头 |

### 1.3 间距 / 圆角 / 阴影

- **间距**：8px 基准（4 / 8 / 12 / 16 / 24 / 32）；表单项垂直间距 20px；卡片内边距 24px。
- **圆角**：按钮 6px、卡片/面板 8px、标签 4px（≤8px）。
- **阴影**：克制——卡片 `0 1px 2px rgba(43,42,40,0.04)`；弹窗 `0 4px 16px rgba(43,42,40,0.08)`；不做大面积投影。
- **留白**：登录/注册居中卡片宽 ~400px；后台内容区左右留白 ≥24px。

### 1.4 组件基线（Element Plus）

`el-form` / `el-input` / `el-button`（`type=primary` 映射陶土色）/ `el-table` / `el-pagination` /
`el-select` / `el-tag`（状态/severity）/ `el-skeleton`（加载）/ `el-empty`（空态）/ `el-alert`（错误条）/
`el-message`（操作反馈）/ `el-message-box`（确认弹窗）/ `el-dropdown`（顶部用户菜单）。

## 二、基础布局（AppLayout）

> 受保护页面的外壳：顶部导航 + 当前用户 + 角色标识 + 登出；权限感知（admin 才显示管理项）。
> 字段来源：`GET /api/auth/me/`（§7）。

```
┌──────────────────────────────────────────────────────────────────────┐
│  ▢ Hermes 测试平台        测试用例                 [admin▾] 主人 ⟲登出  │  ← 顶部导航条（高 56px，白底，底部 1px 边框）
├──────────────────────────────────────────────────────────────────────┤
│                                                                        │
│   <页面内容区 router-view，背景 --bg-base，左右内边距 24px>             │
│                                                                        │
└──────────────────────────────────────────────────────────────────────┘
```

| 元素 | 字段来源 | 说明 | 状态/权限态 |
| --- | --- | --- | --- |
| 品牌区 | 静态 | 左上 logo + "Hermes 测试平台" | 始终显示 |
| 导航项「测试用例」 | 路由 | 跳 `/test-cases`，当前路由高亮（陶土色下划线） | 登录后显示 |
| 当前用户 | `me.username` | 顶部右侧显示用户名（示例"主人"占位） | 登录后显示 |
| 角色标识 | `me.role` | `el-tag`：admin → 陶土 plain 标签"管理员"；member → 灰标签"成员" | 登录后显示 |
| 登出 | — | 调 `POST /api/auth/logout/`，清 token 跳 `/login` | 登录后显示 |

- **未登录**：不渲染布局受保护区，路由守卫直接重定向 `/login`（对应 AC4.2 / TC-ME-002）。
- **响应式**：≥1024px 顶部横向导航；<768px 导航项收进汉堡菜单（第一版仅需保证可用，不强制移动端优化）。

## 三、登录页 `/login`

> 字段来源：`POST /api/auth/login/`（§7）。无需布局外壳，独立居中卡片。

```
                ┌────────────────────────────────────┐
                │            Hermes 测试平台           │   H1 居中，陶土色
                │        登录以访问测试用例平台         │   辅助灰字
                │                                    │
                │  用户名                             │
                │  ┌──────────────────────────────┐  │   el-input
                │  │ tester_new                   │  │
                │  └──────────────────────────────┘  │
                │  密码                               │
                │  ┌──────────────────────────────┐  │   el-input type=password
                │  │ ••••••••                     │  │
                │  └──────────────────────────────┘  │
                │  ┌──────────────────────────────┐  │
                │  │            登 录              │  │   主按钮（陶土色，块级）
                │  └──────────────────────────────┘  │
                │            还没有账号？去注册        │   陶土色链接 → /register
                └────────────────────────────────────┘
```

| 元素 | 组件 | 校验/来源 | 反馈 |
| --- | --- | --- | --- |
| 用户名 | `el-input` | 必填 | 失焦/提交校验，行内红字 |
| 密码 | `el-input[password]` | 必填 | 同上 |
| 登录按钮 | `el-button[primary]` | — | 提交中 `loading` 且禁用，防重复提交 |
| 去注册 | 文本链接 | — | 跳 `/register` |

**状态态覆盖**

| 状态 | 呈现 | 对应用例 |
| --- | --- | --- |
| 正常 | 成功存 token（Pinia + localStorage）→ 跳 `/test-cases` | AC2.1 / TC-LOGIN-001 |
| 加载 | 按钮 loading、表单禁用 | — |
| 错误 400（1101） | 表单上方 `el-alert` 或顶部 `el-message`：**"用户名或密码错误"**（用户名不存在与密码错均同文案，防枚举） | AC2.2 / TC-LOGIN-002,003 |
| 错误 403（1102） | 醒目警告条（warning 黄）：**"账号待管理员审批"** | AC2.3 / TC-LOGIN-004 |
| 错误 403（1103） | 醒目错误条（danger 红）：**"账号已停用，请联系管理员"** | AC2.4 / TC-LOGIN-005 |
| 字段缺失 | 行内字段校验错误 | TC-LOGIN-006 |
| 已登录访问 | 路由守卫重定向 `/test-cases`（§六） | §8 路由守卫 |

## 四、注册页 `/register`

> 字段来源：`POST /api/auth/register/`（§7）。独立居中卡片。

```
                ┌────────────────────────────────────┐
                │             注册新账号               │   H1 居中
                │      注册后需管理员审批方可登录       │   辅助灰字（预告 pending）
                │  用户名 *  ┌────────────────────┐    │   3–150 字符
                │  邮箱   *  ┌────────────────────┐    │   合法邮箱
                │  密码   *  ┌────────────────────┐    │   ≥8，含 Django 校验
                │  确认密码* ┌────────────────────┐    │   == 密码
                │  ┌──────────────────────────────┐  │
                │  │            注 册              │  │   主按钮（陶土色）
                │  └──────────────────────────────┘  │
                │            已有账号？去登录          │   → /login
                └────────────────────────────────────┘
```

| 字段 | 组件 | 前端校验 | 后端错误码映射 |
| --- | --- | --- | --- |
| username | `el-input` | 必填，3–150 | 1001 用户名已被占用 → 行内"用户名已被占用" |
| email | `el-input` | 必填，邮箱格式 | 1002 邮箱已被占用 → 行内"邮箱已被占用" |
| password | `el-input[password]` | 必填，≥8 | 1004 弱密码 → 行内显示后端具体校验信息 |
| password_confirm | `el-input[password]` | 必填，== password（前端先校） | 1003 两次密码不一致 → 行内"两次密码不一致" |

**状态态覆盖**

| 状态 | 呈现 | 对应用例 |
| --- | --- | --- |
| 正常 201 | 成功提示（success 绿 `el-message`）**"注册成功，待管理员审批"** → 1.5s 后引导回 `/login`（或显示"去登录"按钮） | AC1.1 / TC-REG-001 |
| 加载 | 按钮 loading + 表单禁用 | — |
| 字段级错误（1001/1002/1003/1004） | 对应字段下方红字行内提示，后端 `errors` 字段映射到对应输入框 | TC-REG-002~007 |
| 边界 | 用户名 2/151、密码 7 等触发前端+后端校验，行内提示 | TC-REG-008,009 |

> **交互要点**：前端先做 `password == password_confirm`、邮箱格式、长度校验，减少无效请求；但**最终以后端业务码为准**（前端不做权限/唯一性判定）。

## 五、用例展示页 `/test-cases`

> 字段来源：`GET /api/testcases/`（§7）；admin 同步 `POST /api/testcases/sync/`。受 AppLayout 外壳保护。

```
┌──────────────────────────────────────────────────────────────────────┐
│  测试用例                                          [ ⟳ 同步用例 ](admin) │  H1 + 右上同步按钮(仅admin)
│  ┌────────────────────────────────────────────────────────────────┐  │
│  │ 模块 [全部 ▾]   关键词 [____________] 🔍   [重置]                  │  │  筛选区
│  └────────────────────────────────────────────────────────────────┘  │
│  ┌────────────────────────────────────────────────────────────────┐  │
│  │ 模块      │ 标题            │ node_id              │ story │severity│  │  表头
│  ├───────────┼─────────────────┼──────────────────────┼───────┼────────┤  │
│  │ login     │ 正常登录返回token│ test_case/..::test_ok│ 登录  │critical│  │
│  │ login     │ 错误密码统一提示 │ ...::test_wrong_pwd  │ 登录  │ normal │  │
│  │ testcase  │ 同步解析入库     │ ...::test_sync       │ 同步  │ blocker│  │
│  └───────────┴─────────────────┴──────────────────────┴───────┴────────┘  │
│                                  共 128 条   < 1 2 3 ... 7 >  20/页 ▾     │  分页
└──────────────────────────────────────────────────────────────────────┘
```

### 5.1 筛选区

| 元素 | 组件 | 参数（§7） | 说明 |
| --- | --- | --- | --- |
| 模块筛选 | `el-select` | `module_key` | 选项来自现有数据的模块去重；含"全部"=不传 |
| 关键词搜索 | `el-input` + 搜索 | `keyword` | 匹配 node_id/case_title/function_name；回车或点🔍触发 |
| 重置 | `el-button[plain]` | — | 清空筛选，回到默认列表 |

### 5.2 表格列（字段来源 §7 results[]）

| 列 | 字段 | 说明 |
| --- | --- | --- |
| 模块 | `module_key`（hover 显示 `module_name`） | 第一版二者同值 |
| 标题 | `case_title` | 空值显示 `—` 灰字 |
| node_id | `node_id` | 等宽字体，过长省略 + tooltip |
| story | `story` | 空值 `—` |
| severity | `severity` | `el-tag` 按 §1.1 severity 配色；空值 `—` |

> 分页参数：`page`（默认 1）、`page_size`（默认 20，可选 20/50/100，上限 100）。

### 5.3 状态态覆盖

| 状态 | 呈现 | 对应用例 |
| --- | --- | --- |
| 正常 | 表格 + 分页正常渲染，默认按 module_key,node_id 排序 | AC7.1 / TC-LIST-001 |
| 加载 | `el-skeleton` 表格骨架（TanStack Query `isLoading`） | — |
| 空（无数据） | `el-empty`："暂无用例，请管理员同步"；admin 额外显示"立即同步"引导 | §8 / TC-LIST-006 |
| 空（筛选无结果） | `el-empty`："没有符合条件的用例"，提供"重置筛选" | TC-LIST-006 |
| 错误 | `el-alert`（danger）+ "重试"按钮（重新发起查询） | — |
| 未登录 | 路由守卫重定向 `/login`（请求层 401 兜底登出） | AC7.4 / TC-LIST-004 |
| 权限态 | "同步用例"按钮**仅 admin 可见可用**（member 不渲染该按钮） | §8 / AC6.3 |

### 5.4 admin「同步用例」交互（确认 → 提交中 → 成功/失败）

> 呼应 `UI/AGENTS.md`"重试类操作必须有确认、提交中、成功、失败反馈"。

| 阶段 | 呈现 |
| --- | --- |
| 触发 | 点"同步用例" → `el-message-box` 确认弹窗："将重新扫描 api-test 用例并更新清单，是否继续？" |
| 提交中 | 按钮 loading + 文案"同步中…"，禁用重复点击 |
| 成功 | `el-message`（success 绿）："同步完成：扫描 N，新增 X，更新 Y，下线 Z"（来自 `data.scanned/created/updated/deactivated`）→ 自动刷新表格（AC6.1 / TC-SYNC-001） |
| 失败-无权限 | 理论上 member 不可见此按钮；若后端返 403（1201）→ `el-message`（danger）："无权限执行同步"（AC6.3 / TC-SYNC-003） |
| 失败-路径异常 | 后端返 400（1202）→ `el-message`（danger）："用例根目录不存在或不可读，请检查 API_TEST_ROOT 配置"（AC6.4 / TC-SYNC-004） |

## 六、路由与全局交互

| 规则 | 行为 | 对应 |
| --- | --- | --- |
| 路由守卫（未登录） | 访问 `/test-cases` 等受保护路由且无有效 token → 重定向 `/login` | §8 / AC7.4 |
| 路由守卫（已登录） | 已有 token 访问 `/login` 或 `/register` → 重定向 `/test-cases` | §8 |
| Axios 请求拦截 | 自动附加 `Authorization: Token <token>` | §8 |
| Axios 响应拦截 | 收到 401 → 清 token + 跳 `/login`（token 失效兜底，呼应 AC3.1 登出后旧 token 失效） | AC3.1 / AC4.2 |
| 登出 | 顶部"登出" → `POST /api/auth/logout/` → 清 token → 跳 `/login` | AC3.1 / TC-LOGOUT-001 |

## 七、响应式与可访问性

- **断点**：≥1024px 标准后台布局；768–1024px 内容区自适应、表格横向滚动；<768px 顶部导航折叠（第一版保证可用即可）。
- **表格**：列宽自适应，node_id 列允许横向滚动 + tooltip，避免撑破布局。
- **可访问性**：表单 label 关联输入、错误用红色 + 文字（不只靠颜色）；按钮可键盘聚焦；状态标签带文字而非纯色块。

## 九、安全与容器化

- 不出现真实账号、密码、token、生产 URL；示例数据均为占位（如 `tester_new`、`example.test`）。
- API 基址走环境变量（`VITE_API_BASE_URL`），不写死 `localhost`/宿主机端口；报告/静态资源走相对路径或后端授权入口（本需求暂无报告入口）。
- 前端只做体验层权限保护（隐藏 admin 按钮），**真正权限判定在后端**（同步接口后端校验 `IsAdminRole`）。

## 十、给前端阶段的 Playwright 测试线索（前瞻，不在本阶段实现）

- 登录：active 登录成功跳转；pending/disabled 显示对应文案；错误密码统一提示。
- 注册：成功提示待审批；重复用户名/邮箱、两次密码不一致行内报错。
- 用例展示：列表渲染、模块筛选、关键词搜索、空状态、分页；member 不见同步按钮、admin 可见并完成同步反馈。
- 路由守卫：未登录访问 `/test-cases` 跳 `/login`；登录后访问 `/login` 跳 `/test-cases`。

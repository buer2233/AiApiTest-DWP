# Stage 9 Visual Philosophy

## Quiet Operations

Stage 9 uses a quiet operational aesthetic: warm canvas, restrained hairlines, dense tables, and one dark product surface that gives the page a clear center of gravity. The interface should feel like a professional testing console rather than a landing page. Information is expressed through spacing, table rhythm, status pills, and a small number of decisive actions.

The visual system follows the installed Claude design reference: cream canvas, coral primary actions, deep product panels, and serif display type only where hierarchy benefits from it. The craftsmanship target is calm precision. Every repeated row, filter, tab, and dialog control should look intentionally placed, not decorated.

The reference screenshots influence structure rather than business content. Stage 9 keeps the screenshot concepts that matter for a generic platform: module pass-rate table, filters, failed-case dialog, selected retry, all-failed retry, module retry, Jenkins entry, and Allure report entry. It avoids company-specific labels and preserves portable test-platform language.

Data surfaces are intentionally compact. Tables may scroll horizontally on mobile because the information is inherently tabular; controls and hero actions wrap before they compress into unreadable text. This keeps the page usable for repeated QA workflows while preserving the Claude-style warmth and restraint.
